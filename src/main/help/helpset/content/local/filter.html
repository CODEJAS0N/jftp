Setting up Filters

Another useful feature of JFTP is the ability to filter local files when performing various tasks. You can filter local files based on their name pattern, by their modification date and/or by their attributes. The filter will be honored by all local file tasks such as browsing, uploading, deleting, zipping and viewing properties.

Below are a few scenarios where filters can be very handy:

  • Assume that, you have modified quite a few pages on your web site and you know that you have modified only HTML files. In this case, you can set up a name filter that allows only HTML files and upload your entire web site with out having to pick the modified files in various folders. When you do the upload, JFTP recurs through your web site and uploads only HTML files ignoring other files such as images, movies etc. thus, saving you time and bandwidth.
  • Another example is if you want to upload all files that were modified today. In this case you can set up a date filter that allows only those files whose modification date is today's date.
  • Or even better, you can set up a filter that combines both name and date filters.

To set up a filter on the local files, do any one of the following:

  • Select Local File -> Apply Filter... menu item from the menu bar
  • Right click (Ctrl + click on Mac OS X) any where in the local pane and select Apply Filter... from the context menu

When you do any of the above actions, a new dialog titled Filter Local Files will be displayed.

Filter by Name

Regular Expression
The name filter allows you to specify a filter based on the file names or name patterns. You can specify any valid regular expression as your filter criteria.

Below are a few commonly used regular expressions:

  • .*\.html matches all files with extension html.
  • n.* matches all files that start with letter 'n'.
  • .*\.html|.*\.jsp matches all files with extensions html and jsp.
  • index.html matches all files whose name is index.html.

Case Sensitive
Checking this check box does a case sensitive pattern matching. This means that the regular expression .*\.html will match index.html but not INDEX.HTML.

Filter by Date

The date filter allows you to see only those files whose modification date is in a given date range.

Start
Start date for this filter.

End
End date for this filter.

If you want to set up a filter that allows only files modified after a specific date, just specify the Start date and leave the End date blank. Similarly, to allow files that were modified before a specific date, leave the Start date blank and specify the End date.

Filter by Attributes

Show Hidden Files and Folders
Check this box to allow hidden files and folders pass through this filter.

All newly opened sessions will have a default filter that hides hidden files and folders. If you want to see all hidden files and folders, select Local System -> Clear Filter menu item from the menu bar.

Apply
Clicking this button will validate your filter criteria and sets up a filter. At this point, all FTP and local file tasks that you perform will honor the filter.

Cancel
Clicking this button discards any changes you have made to the filter criteria and closes the Filter Local Files dialog.
